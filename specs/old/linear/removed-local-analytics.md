# Removed: Local analytics DB
> Inventory of all non-Linear local SQLite functionality being removed in this migration. This data lived in `.pm/tasks.db` alongside the task tables but was not project management — it was session analytics, observability, and framework versioning.

## Tables removed

### `workflow_events`
Tool calls, skill invocations, phase transitions, subagent spawns. Populated by JSONL transcript parsing.

```sql
CREATE TABLE workflow_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    session_id TEXT,
    sprint TEXT,
    task_num INTEGER,
    event_type TEXT NOT NULL,       -- tool_call, subagent_spawn, phase_entered,
                                    -- task_completed, skill_invoked, session_start, session_end
    tool_name TEXT,                 -- Read, Edit, Write, Bash, Task, Skill, Glob, Grep...
    tool_use_id TEXT,
    skill_name TEXT,
    skill_version TEXT,
    phase TEXT,                     -- RED, GREEN, REFACTOR, AUDIT, etc
    agent_id TEXT,
    agent_type TEXT,                -- Explore, Plan, Bash, claude-code-guide, etc
    input_tokens INTEGER,
    output_tokens INTEGER,
    tool_calls_in_msg INTEGER,
    metadata TEXT                   -- JSON blob
);
```

### `transcripts`
Index of Claude Code JSONL session files. Maps sessions to tasks, tracks token usage, cost estimates, cache stats, error counts, compaction events.

```sql
CREATE TABLE transcripts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    parent_session_id TEXT,
    agent_id TEXT,
    file_path TEXT NOT NULL,
    file_size_bytes INTEGER,
    message_count INTEGER,
    user_message_count INTEGER,
    assistant_message_count INTEGER,
    tool_call_count INTEGER,
    total_input_tokens INTEGER,
    total_output_tokens INTEGER,
    estimated_cost_usd REAL,
    model TEXT,
    started_at DATETIME,
    ended_at DATETIME,
    sprint TEXT,
    task_num INTEGER,
    user_message_timestamps TEXT,       -- JSON array
    cache_read_tokens INTEGER DEFAULT 0,
    cache_creation_tokens INTEGER DEFAULT 0,
    total_user_content_length INTEGER DEFAULT 0,
    stop_reason_counts TEXT,            -- JSON
    thinking_message_count INTEGER DEFAULT 0,
    thinking_total_length INTEGER DEFAULT 0,
    service_tier TEXT,
    has_sidechain INTEGER DEFAULT 0,
    system_error_count INTEGER DEFAULT 0,
    system_retry_count INTEGER DEFAULT 0,
    avg_turn_duration_ms INTEGER,
    tool_result_error_count INTEGER DEFAULT 0,
    compaction_count INTEGER DEFAULT 0,
    cwd TEXT,
    git_branch TEXT,
    assistant_message_timestamps TEXT,  -- JSON array
    background_task_count INTEGER DEFAULT 0,
    web_search_count INTEGER DEFAULT 0,
    synced_at DATETIME,
    sync_attempts INTEGER DEFAULT 0,
    sync_error TEXT
);
```

### `messages`
Full content of every conversation message (no truncation).

```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    message_index INTEGER NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
    content TEXT,
    timestamp TEXT,
    model TEXT,
    input_tokens INTEGER,
    output_tokens INTEGER,
    stop_reason TEXT,
    synced_at DATETIME,
    UNIQUE (session_id, message_index)
);
```

### `tool_calls`
Full input params for every tool invocation.

```sql
CREATE TABLE tool_calls (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    message_index INTEGER NOT NULL,
    tool_index INTEGER NOT NULL,
    tool_use_id TEXT,
    tool_name TEXT NOT NULL,
    input TEXT NOT NULL,  -- JSON string
    output TEXT,
    is_error INTEGER DEFAULT 0,
    timestamp TEXT,
    synced_at DATETIME,
    UNIQUE (session_id, message_index, tool_index)
);
```

### `developer_context`
Session-level developer state for analytics (concurrent sessions, time of day, etc.).

```sql
CREATE TABLE developer_context (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    developer TEXT NOT NULL,
    recorded_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    concurrent_sessions INTEGER DEFAULT 1,
    hour_of_day INTEGER,
    alertness REAL,
    environment TEXT,
    notes TEXT
);
```

### `skill_versions`
Version history per skill, populated during framework sync.

```sql
CREATE TABLE skill_versions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_name TEXT NOT NULL,
    version TEXT NOT NULL,
    framework_version TEXT,
    introduced_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    changelog TEXT,
    UNIQUE(skill_name, version)
);
```

### `event`
Local event log for N2O API sync. Events generated by `n2o issue *` commands (formerly `n2o task *`), flushed on `n2o sync`.

```sql
CREATE TABLE event (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id TEXT NOT NULL UNIQUE,
    event_type TEXT NOT NULL,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    user_id TEXT,
    project_id TEXT,
    payload TEXT NOT NULL,     -- JSON blob
    synced_at DATETIME,
    sync_attempts INTEGER DEFAULT 0,
    sync_error TEXT
);
```

## Views removed

### Task-adjacent analytics (depended on `tasks` table)
- `needs_pattern_audit` — green tasks not yet audited
- `needs_verification` — green + audited tasks pending PM verification
- `refactor_audit` — tasks with skill update notes
- `velocity_report` — task completion times
- `sprint_velocity` — average completion time per sprint
- `developer_velocity` — avg minutes per task by person
- `estimation_accuracy` — estimate vs actual by person
- `estimation_accuracy_by_type` — estimate vs actual by task type
- `estimation_accuracy_by_complexity` — estimate vs actual by complexity
- `developer_quality` — reversions and testing posture by person
- `developer_learning_rate` — estimation accuracy trend by sprint
- `common_audit_findings` — audit problems by developer
- `reversion_hotspots` — which task types cause the most trouble
- `blow_up_factors` — tasks where actual > 2x estimated
- `token_efficiency_trend` — avg tokens per task by sprint and complexity

### Session/skill analytics (depended on `workflow_events` / `transcripts`)
- `skill_usage` — frequency of each tool/skill across sessions
- `phase_timing` — how long each TDD phase takes
- `skill_token_usage` — per-skill token totals by sprint
- `skill_duration` — duration per skill invocation
- `skill_precision` — files read vs modified per task
- `phase_time_distribution` — phase duration as % of total task time
- `skill_version_token_usage` — token usage by skill version
- `skill_version_duration` — duration by skill version
- `skill_version_precision` — exploration ratio by skill version
- `task_trajectory` — phase sequence and audit reversions per task
- `brain_cycles_per_task` — user messages as proxy for decision loops
- `context_loading_time` — exploration before first edit
- `session_health` — per-session errors, retries, compaction

### Concurrency views (depended on `transcripts` / `tasks`)
- `concurrency_agents` — running count of all agents over time
- `concurrency_sessions` — running count of primary sessions
- `concurrency_tasks` — overlapping task work windows

## Triggers removed

- `update_task_timestamp` — auto-update `updated_at` on task modification
- `set_started_at` — auto-set when task leaves pending
- `set_completed_at` — auto-set when task reaches green
- `track_reversion` — increment reversions on backward status change

## CLI code removed (analytics-related)

### `cli/db/`
Entire package — `Open()`, `AutoMigrate()`, `InitFromSchema()`, `InitFromSchemaBytes()`. No more local SQLite.

### `cli/sync/sync.go`
- `ExtractSkillVersions()` — wrote to `skill_versions` table. Drop.
- `parseSkillFrontmatter()`, `parseYAMLLine()` — only used by `ExtractSkillVersions`. Drop.
- `SyncDirectory()`, `FileChecksum()`, `copyFile()` — keep if still used by framework file sync, otherwise drop.

### `cli/event/event.go`
Needs review — if it only supports the event log table, delete entirely.

## Schema file

`.pm/schema.sql` — delete entirely. No local DB to initialize.

## Why this is being removed

1. **Not being used.** Most of these tables and views are aspirational — the analytics pipeline to populate them (JSONL parsing, transcript collection) is incomplete or unused.
2. **Wrong layer.** Session analytics (token usage, tool calls, thinking time) are Claude Code concerns, not PM concerns. If needed, they belong in a separate observability tool, not baked into the workflow CLI.
3. **Simplification.** Removing SQLite entirely means dropping `modernc.org/sqlite` as a dependency, eliminating DB initialization from `n2o init`, and removing migration infrastructure. The CLI becomes a pure API client.
4. **If needed later**, session analytics can be rebuilt as a standalone tool that reads Claude Code's JSONL files directly, without coupling to the workflow system.
